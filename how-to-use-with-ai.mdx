Most tools in this toolkit are text protocols, behavioral contracts, and operational frameworks — not software packages you install. They are meant to be loaded into an AI assistant's context and used as structured guidance for real work. The fastest way to get value from them is to work with your AI, not just read the documentation and try to figure out the system alone.

## Getting Started

<Steps>
  <Step title="Find the right tool for your problem">
    Browse the sidebar to locate the category that matches your situation — behavior shaping, session management, project setup, or infrastructure. Read the description before committing to a specific tool. Many of them overlap, and picking the wrong one wastes time.
  </Step>
  <Step title="Open the tool's repository on GitHub">
    Every tool in this collection has a source repository at [github.com/XxYouDeaDPunKxX](https://github.com/XxYouDeaDPunKxX). Open the repo directly to read the raw files — the README, protocol files, and any supporting documents. This is the actual material your AI will work with.
  </Step>
  <Step title="Paste the repository link into your AI chat">
    Drop the repository URL into your AI assistant and ask what it does. Let the AI read and summarize before you start asking it to apply anything.

    ```text
    What is this project for, how does it work, and what could I use it for in my own workflow?
    ```
  </Step>
  <Step title="Ask for practical examples and probe the limits">
    Once the AI understands the tool, push further. Ask for concrete examples of how to apply it, what situations it handles poorly, and whether it actually fits your current problem. A tool that almost fits is often worse than no tool at all.
  </Step>
  <Step title="Adapt it to your own context">
    These tools were built for MaRCoRe's workflows first. They will not fit yours exactly. Read the protocol, understand the intent behind each rule or gate, and modify it to match the way you actually work. The goal is operational value, not compliance with someone else's system.
  </Step>
</Steps>

<Tip>
  When you paste a repo link into your AI, ask these questions explicitly: Which files should I read first? What are the limits of this tool? Does it actually fit my problem, or is something else in this collection a better match? A few targeted questions up front save significant time later.
</Tip>

## What These Tools Are NOT

Before investing time in any protocol here, be clear about what you are working with:

- **Not npm packages.** Nothing here installs with a command. There is no dependency tree, no runtime, no build step.
- **Not plug-and-play.** Dropping a protocol file into a chat without reading it first will produce shallow results. These systems require understanding to apply correctly.
- **Not universal solutions.** Each tool was built for a specific class of problem. Using a session management protocol when you need a behavioral contract — or vice versa — will not help.
- **Not finished products.** Some are rough. Some are still being shaped. All of them require you to read, adapt, and decide what fits.

## Why This Approach Works

These tools were built from real use — from sessions that drifted, decisions that got lost, and AI outputs that quietly stopped being useful. The protocols exist because hoping AI assistants stay on track does not work. Loading explicit behavioral contracts, session structure, and review gates into context does.

The same principle applies to learning them: passive reading produces limited results. Active use with your AI assistant — loading, questioning, adapting — is how these tools deliver value.
