ChatGPT Local Agent MCP is an MCP (Model Context Protocol) server that runs on your Windows PC and connects ChatGPT to your local environment. Unlike browser-only ChatGPT, this lets the model help with real files, run commands, take screenshots, and interact with desktop tools — while you remain in control of what it can and can't access.

## Architecture Overview

The local agent runs as an MCP server on your Windows machine. ChatGPT connects to it using the MCP protocol, and you configure exactly what the server exposes. Nothing passes through an external relay.

- **Local MCP server** runs on your Windows machine
- **ChatGPT connects** to it via the MCP protocol
- **You control** what the server exposes — files, commands, browser, screenshots
- **All operations happen locally** — nothing is sent to an external relay

<Note>
  MCP (Model Context Protocol) is an open protocol for connecting AI models to local environments and tools.
</Note>

  <Step title="Clone the repository">

  </Step>
  <Step title="Follow the setup instructions">
    Read the repository README and complete the setup steps for your environment.
  </Step>
  <Step title="Configure exposed operations">
    Decide which operations — file access, command execution, browser, screenshots — you want to expose to ChatGPT, and configure the server accordingly.
  </Step>
  <Step title="Start the local MCP server">
    Launch the server on your Windows machine using the command provided in the README.
  </Step>
  <Step title="Connect ChatGPT">
    Connect ChatGPT to the local MCP server using the connection details provided by the server on startup.
  </Step>
  <Step title="Test the connection">
    Verify the setup with a simple file read request to confirm ChatGPT can reach your local environment.
  </Step>
</Steps>
